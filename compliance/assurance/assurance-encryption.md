## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="7c5a5-117">Comment Microsoft 365 chiffre-t-il les données en transit ?</span><span class="sxs-lookup"><span data-stu-id="7c5a5-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="7c5a5-118">Les produits et services Microsoft 365 utilisent des protocoles de transport forts, tels que TLS, pour empêcher les personnes non autorisées d’espionner des données client lors de leur déplacement sur un réseau.</span><span class="sxs-lookup"><span data-stu-id="7c5a5-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="7c5a5-119">Les messages électroniques en cours de livraison, les conversations en cours dans une réunion en ligne ou les fichiers répliqués entre des centres de données sont des exemples de données en transit.</span><span class="sxs-lookup"><span data-stu-id="7c5a5-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="7c5a5-120">Dans Microsoft 365, les données sont considérées comme « en transit » chaque fois que l’appareil d’un utilisateur communique avec un serveur Microsoft ou qu’un serveur Microsoft communique avec un autre serveur.</span><span class="sxs-lookup"><span data-stu-id="7c5a5-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="7c5a5-121">Exchange Online, SharePoint Online, Microsoft Teams et Office Online utilisent tous TLS pour s’assurer que les données restent confidentielles pendant leur transit.</span><span class="sxs-lookup"><span data-stu-id="7c5a5-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="7c5a5-122">Comment Microsoft 365 gère-t-il les clés utilisées pour le chiffrement ?</span><span class="sxs-lookup"><span data-stu-id="7c5a5-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="7c5a5-130">Les clients ayant besoin de contrôler leurs propres clés de chiffrement racine peuvent tirer parti du chiffrement de service avec la clé client.</span><span class="sxs-lookup"><span data-stu-id="7c5a5-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="7c5a5-131">À l’aide de la clé client, les clients peuvent générer leurs propres clés de chiffrement à l’aide d’un module de service matériel (HSM) local ou d’Azure Key Vault (AKV).</span><span class="sxs-lookup"><span data-stu-id="7c5a5-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="7c5a5-132">Les clés racine du client sont stockées dans AKV, où elles peuvent être utilisées comme racine de l’un des chaînes de clés qui chiffre les données ou les fichiers de boîte aux lettres du client.</span><span class="sxs-lookup"><span data-stu-id="7c5a5-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="7c5a5-133">Les clés racine client ne sont accessibles qu’indirectement par le code de service Microsoft 365 pour le chiffrement des données et ne sont pas accessibles directement par les employés de Microsoft.</span><span class="sxs-lookup"><span data-stu-id="7c5a5-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
